## Целевые значения (target)

Каждый пример имеет 3 оценки качества в диапазоне [0, 1]:

1. **Correctness (правильность)** - насколько код корректен и работоспособен
2. **Efficiency (эффективность)** - насколько эффективен алгоритм
3. **Readability (читаемость)** - насколько код понятен и читаем

### Градация качества:

- **Отличное** (0.85 - 1.0): правильный, эффективный, читаемый код
- **Хорошее** (0.65 - 0.85): работающий код с небольшими недостатками
- **Среднее** (0.45 - 0.65): код работает, но неоптимален или нечитаем
- **Плохое** (0.0 - 0.45): код с ошибками или очень плохого качества

## Нормализация данных

### Математическое обоснование

Перед подачей в нейронную сеть признаки нормализуются для обеспечения:
1. **Единого масштаба** всех входных данных (избежание доминирования признаков с большими значениями)
2. **Стабильности обучения** (предотвращение проблем с градиентами)
3. **Ускорения сходимости** (быстрое достижение оптимума)

### Формула нормализации

Для каждого признака применяется линейное масштабирование:

```
x_normalized = x_original / scale_factor
```

Где `scale_factor` выбран исходя из ожидаемого максимального значения признака в реальных данных.

### Примеры нормализации

```python
# В train_neural_network.py и train_final_model.py:
feature_vector = [
    features['lines_of_code'] / 100.0,      # 0-100 строк → [0, 1]
    features['functions_count'] / 10.0,     # 0-10 функций → [0, 1]
    features['complexity'],                  # уже в [0, 1]
    features['nested_levels'] / 5.0,        # 0-5 уровней → [0, 1]
    features['variable_names_length'] / 20.0,  # 0-20 символов → [0, 1]
    features['comments_ratio'],              # уже в [0, 1]
    features['imports_count'] / 10.0,       # 0-10 импортов → [0, 1]
    features['class_count'] / 5.0,          # 0-5 классов → [0, 1]
    features['error_handling'],              # бинарный: 0 или 1
    features['test_coverage']                # уже в [0, 1]
]
```

Результат: все признаки находятся в диапазоне **[0, 1]**, что оптимально для сигмоидной и ReLU активации.

### Пример 2: Средний код

```python
def sort_list(numbers):
    result = []
    for i in range(len(numbers)):
        min_idx = i
        for j in range(i+1, len(numbers)):
            if numbers[j] < numbers[min_idx]:
                min_idx = j
        numbers[i], numbers[min_idx] = numbers[min_idx], numbers[i]
    return numbers
```

**Оценки:** correctness=0.8, efficiency=0.6, readability=0.6

**Почему средне:**
- Работает корректно
- Но сложность O(n²) вместо O(n log n)
- Нет комментариев
- Избыточный код

## Математическое обоснование нормализации признаков

### Зачем нормализация?

Нейронная сеть работает лучше, когда все входные данные находятся в одном диапазоне (обычно [0, 1] или [-1, 1]).

### Формула нормализации:

```
x_normalized = x_raw / x_max
```

где:
- `x_raw` - исходное значение признака
- `x_max` - максимальное ожидаемое значение
- `x_normalized` - нормализованное значение ∈ [0, 1]

### Примеры:

1. **lines_of_code** / 100.0
   - Ожидаем: код до 100 строк
   - Пример: 25 строк → 25/100 = 0.25

2. **complexity** / 10.0
   - Ожидаем: сложность до 10
   - Пример: сложность 3 → 3/10 = 0.3

3. **comments_ratio** (уже в [0, 1])
   - Процент строк с комментариями
   - Пример: 10% комментариев → 0.1
